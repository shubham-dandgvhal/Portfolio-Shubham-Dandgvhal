Weather Dashboard

A responsive weather app that lets users search any city and see the current conditions, a five-day outlook, and sunrise/sunset times at a glance. Built with React and a public weather API.

RoleSolo developer
TimelineThree weeks
TypeReact single-page app

The idea

Most weather sites are either too busy or too sparse. I wanted a middle-ground dashboard that surfaces the information I actually care about in one scroll, without ads or filler.

How it works

  • Typing a city fetches current conditions and a forecast from the API.
  • Responses are cached in memory so switching tabs stays snappy.
  • Units can be toggled between Celsius and Fahrenheit and the choice persists.
  • Error states (bad city, network failure) show helpful messages instead of spinning forever.

What I learned

  • Working with React hooks for state and effects.
  • Structuring components around data flow instead of visuals.
  • Handling async errors gracefully — the happy path is the easy half.

What's next

Geolocation on first load, saved "favorite" cities, and a small hourly chart for the next twelve hours.
