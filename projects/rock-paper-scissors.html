Rock, Paper, Scissors

A colorful take on the classic schoolyard game, built entirely in vanilla JavaScript. Keeps track of wins, losses, and ties so you can settle the debate for good.

RoleSolo developer
TimelineOne weekend
TypeBrowser game

The idea

The goal was to practice DOM manipulation and event handling with something small enough to finish in a sitting but interactive enough to stay fun. Rock, Paper, Scissors fits the bill — clear rules, a natural click-based UI, and a built-in way to show state.

How it works

Each move is a button. When the player clicks one, the game picks a random counter-move for the CPU, figures out who won, and updates three things: the score, a status message, and an animated reveal of the two choices side-by-side.

  • Score state is kept in a small object and persisted to localStorage.
  • A reset button clears the score and resets the status text.
  • The UI uses CSS transitions so wins feel more satisfying.

What I learned

  • How to split logic (picking a winner) from presentation (rendering a round).
  • Why keeping state in one place makes the UI much easier to keep in sync.
  • Small details — an ARIA live region for the score — go a long way for accessibility.

What's next

I'd like to add a "best of 5" mode and swap the emoji icons for hand-drawn SVGs so the game feels more personal.
